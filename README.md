# face-gesture-unlock

A lightweight biometric login system for Ubuntu using:
1. **Face recognition** (InsightFace ArcFace)
2. **Hand gesture fallback** (MediaPipe — double middle finger)
3. **Password fallback** (standard Ubuntu PAM)

Works with GDM / LightDM via PAM (Pluggable Authentication Modules), similar to Windows Hello.

---

## System Architecture
```
Login triggered
      │
      ▼
[face_unlock.py]  ← called by pam_python.so
      │
      ▼
[auth_controller.py]
      │
      ├─── Open webcam (3 second window)
      │
      ├─── FaceAuthenticator
      │       └── InsightFace ArcFace embedding comparison
      │               ↓ PASS → PAM_SUCCESS → logged in
      │
      ├─── GestureAuthenticator
      │       └── MediaPipe — both hands, middle finger only
      │               ↓ PASS → PAM_SUCCESS → logged in
      │
      └─── FAIL → PAM_AUTH_ERR → password prompt shown
```

## Installation

### 1. Clone the repository
```bash
git clone https://github.com/YOUR_USERNAME/face-gesture-unlock.git
cd face-gesture-unlock
```

### 2. Create a virtual environment
```bash
python3 -m venv venv
source venv/bin/activate
```

### 3. Install Python dependencies
```bash
pip install --upgrade pip
pip install -r requirements.txt
```

> **Note:** `insightface` will download the `buffalo_sc` model (~30 MB) on first run.

### 4. Install pam-python
```bash
sudo apt update
sudo apt install libpam-python
```

## Dataset Creation

### Step 1 — Capture face images

Sit in front of your webcam in good lighting and run:
```bash
python scripts/capture_faces.py --count 50
```

Images are saved to `dataset/user/raw/`. You can also add your own photos manually.

### Step 2 — Prepare the dataset

Normalise filenames:
```bash
python scripts/prepare_dataset.py
```

Images are renamed to `img1.jpg`, `img2.jpg`, …

### Step 3 — Generate embeddings
```bash
python scripts/generate_embeddings.py
```

This saves `data/embeddings.npy`. Re-run this any time you add new images.

python auth/auth_controller.py

## Enabling PAM Authentication

> ⚠️ **Back up your PAM config before editing.** An incorrect PAM file can lock you out.
> Always keep a root terminal open while testing.

### 1. Note the full path to your repo
```bash
pwd
# Example: /home/alice/face-gesture-unlock
```

### 2. Edit the GDM password PAM file
```bash
sudo cp /etc/pam.d/gdm-password /etc/pam.d/gdm-password.bak
sudo nano /etc/pam.d/gdm-password
```

Add the following line **at the very top** of the file, before any other `auth` lines:
```
auth sufficient pam_python.so /home/alice/face-gesture-unlock/pam/face_unlock.py
```

Replace `/home/alice/face-gesture-unlock` with your actual path.

### 3. Ensure the PAM module uses the correct Python

The `pam_python.so` module uses the system Python. If your dependencies are in a virtualenv, edit `pam/face_unlock.py` and change:
```python
_PYTHON = "/home/alice/face-gesture-unlock/venv/bin/python"
```

### 4. Test

Log out and log back in. The webcam will activate briefly:
- Face recognised → instant login
- Double middle finger → instant login
- Neither → password prompt appears as normal

### Reverting PAM (if something goes wrong)
```bash
sudo cp /etc/pam.d/gdm-password.bak /etc/pam.d/gdm-password
```

## Tuning the Similarity Threshold

In `auth/face_auth.py`:
```python
SIMILARITY_THRESHOLD = 0.40
```

- **Increase** (e.g. `0.50`) → stricter, fewer false positives
- **Decrease** (e.g. `0.30`) → more permissive, easier to match in low light

Run `auth_controller.py` in a terminal while logged in to see live similarity scores and calibrate.

## Troubleshooting

| Problem | Solution |
|---|---|
| `FileNotFoundError: embeddings.npy` | Run `generate_embeddings.py` first |
| `Cannot open webcam` | Check `/dev/video0` exists; try `CAMERA_INDEX = 1` |
| InsightFace model not downloading | Check internet connection; model is ~30 MB |
| PAM not calling the script | Verify the full path in `gdm-password` is correct |
| Locked out after PAM edit | Boot recovery mode → restore backup with `cp` |
| Gesture not detected | Ensure both hands are clearly visible, well lit, ~50 cm from camera |
| Face not matching in low light | Add more images in varied lighting; lower threshold slightly |

## Security Notes

- Embeddings are stored locally — nothing is sent to any server.
- The camera runs for only ~3 seconds per login attempt.
- `auth sufficient` means PAM moves on to the next module (password) on failure — you are never locked out by a bug in this code.
- Do not rely on this as your sole security mechanism; use it as convenience, not as a replacement for a strong password.
